# CLAUDE

このプロジェクトでは、以下の運用ルールに従ってコード変更とGit管理を行います。

## Git運用ルール

- 変更を行ったら必ずコミットする。
- コミットメッセージは意味が分かる内容にする。
- コードを変更するたびに、GitHubにプッシュする。
- プッシュ前にローカルでビルドや動作確認を行う。
- プルリクエストを作成する場合は、変更内容と目的を分かりやすく説明する。

## 開発フロー

1. ブランチを作成する。
2. コードを変更する。
3. 変更内容をローカルで確認する。
4. `git add` でステージングする。
5. `git commit -m "変更内容を簡潔に記載"` でコミットする。
6. `git push` でGitHubに反映する。

## 注意事項

- 変更の単位は小さく、レビューしやすい形にする。
- 大きな変更を行う場合は、事前にブランチと目的を明確にする。
- センシティブな情報は`.env`などの環境ファイルに含め、`.gitignore`で管理する。
